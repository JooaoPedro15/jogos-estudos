# Documentacao do Reavaliacao AEDS II

Esta pasta guarda a documentacao de produto e engenharia do novo jogo.

## Arquivos

| Arquivo | Conteudo |
| --- | --- |
| `product-spec.md` | Objetivo, usuario, problema, escopo e criterios de sucesso. |
| `reavaliacao-format.md` | Como o jogo deve imitar o formato observado nas reavaliacoes. |
| `game-mechanics.md` | Mecanicas de campanha, oficina, simulado e progressao. |
| `content-map.md` | Dominios, topicos e materiais de referencia. |
| `adaptive-error-notebook.md` | Como registrar erros e gerar treinos parecidos. |
| `architecture.md` | Arquitetura planejada para implementacao. |
| `testing-strategy.md` | Estrategia de testes automatizados. |
| `implementation-roadmap.md` | Ordem recomendada de construcao. |

## Principio

Toda mecanica deve responder a uma pergunta simples: isso ajuda o aluno a errar
menos em uma questao real de reavaliacao?
